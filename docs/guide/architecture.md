# 总体系统架构

系统建议分为七个逻辑模块。

```text
数据源 AX 2012 / 中间库
  ↓
数据同步与标准化
  ↓
主数据层 + 业务数据层
  ↓
规则中心
  ↓
排产引擎
  ↓
排产结果 / 异常结果 / 规则命中日志
  ↓
计划员查看、调整、确认
  ↓
形成计划版本
```

## 模块划分

| 模块 | 职责 | 阶段引入 |
| --- | --- | --- |
| 数据同步模块 | 从 AX 2012 或中间库获取 MO、产品、路线、客户、状态等数据，并标准化 | 阶段 0 |
| 主数据管理模块 | 维护产线、工段、产品分类、客户分类、工艺路线、UPPH、日历 | 阶段 0~1 |
| 规则管理模块 | 维护订单筛选、优先级、产线映射、专线、产能、日历例外规则 | 阶段 1~3 |
| 排产引擎模块 | 执行筛单、拆工段、匹配产线、计算产能、占用日历、生成结果 | 阶段 1 |
| 异常诊断模块 | 识别缺数据、缺规则、冲突、产能不足、逾期等 | 阶段 1~3 |
| 结果展示模块 | 按订单、产线、日期、工段、异常维度查看排产结果 | 阶段 1~2 |
| 人工干预模块 | 支持锁单、改线、调整日期、停线、加班、异常放行 | 阶段 3~5 |
