# 总体建设原则

## 1. 先简单后复杂

先选择单一、稳定、可验证的场景形成闭环，再扩展 EE/ESG、多工段、专线、日历例外等复杂场景。

## 2. 先规则后优化

第一阶段使用 **规则 + 贪心算法**，确保结果可解释。

后期再引入：

- 负荷均衡
- 换线损耗
- 物料齐套
- 多线并行
- 优化算法

## 3. 先异常可见后自动修复

缺路线、缺 UPPH、无产线、规则冲突等问题，初期应进入异常清单，而不是由系统自动猜测。

## 4. 先正排后正倒结合

初期采用：

```text
按 DlvDate 升序正排
```

并输出：

- 预计完成日
- 是否逾期
- 逾期天数

后续再扩展倒排或正倒结合。

## 5. 先 MO 级后更细粒度

初期最小结果粒度：

```text
MO + 工段 + 日期 + 产线
```

后续再考虑：

- 批次级
- 小时级
- 分批流转
- 多线并行

## 6. 结果必须可解释

每条排产结果需说明：

- 为什么选这个订单
- 为什么选这个工段
- 为什么选这条线
- 为什么排在这个日期
- 为什么逾期或异常
