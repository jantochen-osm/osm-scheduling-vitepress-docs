# 异常处理设计

| 异常大类 | 异常示例 | 处理方式 |
| --- | --- | --- |
| 订单异常 | DlvDate 为空、Qty <= 0、OSM_Category 缺失、Pool 不匹配 | 阻断排产或进入候选异常清单 |
| 工艺异常 | 缺工艺路线、缺工段、OperationSeq 为空、未知 ProcessType | 阻断对应订单或工段排产 |
| 规则异常 | 无候选产线、命中多个互斥专线、试产线被正式单命中 | 阻断或按优先级处理并记录 |
| 产能异常 | 缺 UPPH、日历工时为 0、产线不可用、产能不足 | 阻断或提示人工确认 |
| 结果异常 | 预计完成日超过交期、工段顺序倒置、计划数量不等于订单数量 | 输出逾期/校验异常 |

## 异常处理原则

```text
缺数据不猜测。
规则冲突不静默覆盖。
缺 UPPH 不跳过工段。
逾期不是系统失败，而是需要明确输出的风险。
```
