# 阶段化实现路径总览

系统建设建议从简单场景逐步扩展到多场景。

```text
阶段 0：业务边界与数据底座确认
阶段 1：EE Assembly 单场景 MVP
阶段 2：EE 多工段与完整 EE 规则扩展
阶段 3：ESG 专线场景扩展
阶段 4：日历例外、异常诊断与可解释性增强
阶段 5：人工干预与计划版本管理
阶段 6：高级约束与优化能力扩展
```

## 阶段衔接总览

| 阶段 | 核心能力 | 依赖上一阶段 | 为下一阶段提供 |
| --- | --- | --- | --- |
| 0 | 数据与边界确认 | 无 | 样例数据、字段清单、MVP 范围 |
| 1 | EE Assembly MVP | 阶段 0 数据样本 | 排产引擎骨架、结果表、异常表 |
| 2 | EE 多工段 | 阶段 1 单工段闭环 | 多工段顺序框架、专线优先级 |
| 3 | ESG 专线 | 阶段 2 多工段框架 | OSM EE+ESG 完整场景 |
| 4 | 日历与可解释 | 阶段 3 结果 | 可诊断、可解释、可验收的排产结果 |
| 5 | 人工干预与版本 | 阶段 4 可解释结果 | 日常计划协同能力 |
| 6 | 高级优化 | 阶段 5 稳定业务使用 | 更高质量、更智能的排产能力 |

## 推荐版本路线图

| 版本 | 名称 | 核心能力 |
| --- | --- | --- |
| V0.1 | 数据样板版 | 订单、产线、工艺、产能、日历数据准备 |
| V0.2 | 单工段排产版 | EE Assembly 单场景排产 |
| V0.3 | 多产线选择版 | 3F3~3F6 最早完成选择 |
| V0.4 | 多工段顺序版 | Assembly + Package 等顺序约束 |
| V0.5 | EE 完整规则版 | 1F、3F EE 规则完整覆盖 |
| V0.6 | ESG 专线版 | 4F1、4F2、4F4、4F6 |
| V0.7 | 日历例外版 | 周六、节假日前、月底最后工作日、停线/加班 |
| V0.8 | 异常诊断版 | 缺路线、缺 UPPH、无产线、逾期诊断 |
| V0.9 | 人工干预版 | 锁单、改线、停线、加班、重排 |
| V1.0 | OSM 正式排产版 | OSM 场景稳定运行，可解释、可调整、可版本管理 |
