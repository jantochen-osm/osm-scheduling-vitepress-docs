# 00 阶段总览

## 阶段目标

Stage 0 的目标是确认排产系统的 **业务边界与数据底座**，为 Stage 1 的 EE Assembly 单场景 MVP 提供稳定输入。

这阶段不追求系统功能完整，而是追求：

- 排产范围清楚
- 数据字段清楚
- 样例数据可用
- 规则基线清楚
- 异常口径清楚
- 验收标准清楚

## 阶段输入

Stage 0 需要收集以下输入：

| 输入 | 来源角色 | 用途 |
| --- | --- | --- |
| MO 订单样本 | 计划 / IT / AX | 验证订单筛选与排序 |
| 工艺路线样本 | IE / 工艺 / IT | 验证 ItemId 到工段的拆解 |
| 产线清单 | 生产 / 计划 | 验证产线映射 |
| UPPH 样本 | IE / 生产 | 验证产能计算口径 |
| 工作日历 | 计划 / HR / 生产 | 验证每日可用工时 |
| 业务规则说明 | 计划 / 生产 | 验证 EE / ESG、专线、试产线等规则 |

## 阶段输出

Stage 0 完成后，需要输出：

| 输出物 | 说明 |
| --- | --- |
| 业务边界说明 | 明确排产对象、范围、粒度、初始假设 |
| 字段映射表 | AX / 中间库字段到系统标准字段的映射 |
| 样例数据包 | 用于 Stage 1 测试的订单、路线、产线、UPPH、日历样例 |
| 主数据基线 | 产线、工段、产品、客户、日历的初始主数据 |
| 规则基线 | 订单筛选、Line Group、产线映射、产能、日历规则 |
| 异常字典 | 缺字段、缺路线、缺 UPPH、无产线等异常定义 |
| Stage 1 准入清单 | 判断是否可以进入 EE Assembly MVP 的检查表 |

## 阶段完成定义

Stage 0 完成的最低标准：

```text
1. 能从样例订单中筛选出 EE Assembly 场景订单。
2. 能通过 ItemId 找到至少一个 Assembly 工段。
3. 能找到 3F3~3F6 的产线主数据。
4. 能找到对应 UPPH 或明确缺 UPPH 异常。
5. 能通过日历得到某一天的标准工时。
6. 能手工演算至少 1 条 MO 的日级排产过程。
```

## 不做事项

Stage 0 不做：

- 不开发完整排产引擎
- 不做前端页面
- 不做复杂优化算法
- 不做物料齐套约束
- 不做人工拖拽
- 不做多线并行
- 不做分批流转

## 阶段风险

| 风险 | 影响 | 控制方式 |
| --- | --- | --- |
| 字段口径不统一 | 后续筛单错误 | 建立字段映射表和字段说明 |
| 工艺路线样例不足 | Stage 1 无法验证 | 至少准备 20~50 条跨产品样例 |
| UPPH 口径不清 | 产能计算错误 | 明确线体级 / 人均级 / 产品级 / 工段级 |
| 产线状态不清 | 试产线可能误排 | 建立 isSchedulable / isTrialLine 字段 |
| 日历例外未定义 | 工时计算不一致 | 明确周六、周日、节假日、月底最后工作日规则 |
