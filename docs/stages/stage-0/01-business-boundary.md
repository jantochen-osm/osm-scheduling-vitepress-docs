# 01 业务边界确认

业务边界用于回答：**这个排产系统第一版到底排什么、不排什么、排到什么程度。**

## 1. 排产对象

第一版建议固定为：

```text
MO 制造订单
```

暂不以销售订单、客户订单、工单批次、小时工序作为主排产对象。

## 2. 排产范围

第一版范围：

```text
OSM 产线
```

订单入口条件：

```text
Status != 已结束
AND Pool IN (SC_YBSC_F3, SC_YBSC_HT, SCD_HT_CC, SCD_HT_F3)
```

## 3. 第一版 MVP 场景

Stage 1 目标建议严格收敛到：

```text
OSM_Category = EE
ProcessType = Assembly
Candidate Lines = 3F3, 3F4, 3F5, 3F6
```

这意味着 Stage 0 必须准备支撑该场景的数据。

## 4. 排产粒度

第一版结果粒度：

```text
MO + 工段 + 日期 + 产线
```

示例：

| MO | 工段 | 日期 | 产线 | 数量 |
| --- | --- | --- | --- | --- |
| MO001 | Assembly | 2026-05-13 | 3F3 | 1000 |
| MO001 | Assembly | 2026-05-14 | 3F3 | 800 |

## 5. 初始假设

### 5.1 订单拆分假设

```text
同一 MO 的同一工段允许跨日，不允许跨线。
```

含义：

- 一个 MO 数量大，可以排多天；
- 但同一个工段不会同时排到 3F3 和 3F4。

### 5.2 工段流转假设

```text
前工段整单完成后，后工段才开始。
```

Stage 1 只有 Assembly 单工段，暂时不涉及多工段流转。Stage 2 才引入。

### 5.3 排产方向假设

```text
按 DlvDate 升序正排。
```

系统输出预计完成日，并判断是否逾期。

## 6. 明确不做事项

Stage 0 和 Stage 1 不做：

- ESG 专线排产
- Package 工段
- Cutting 工段精确产能
- 物料齐套约束
- 多线并行
- 分批流转
- 换线损耗
- 复杂优化算法
- 人工拖拽调整

| 问题 | 建议答案 | 是否确认 |
| --- | --- | --- |
| 第一版是否只做 EE Assembly？ | 是 | 已确认 |
| 是否只使用 3F3~3F6？ | 是 | 已确认 |
| 同一 MO 同一工段是否不跨线？ | 是 | 已确认 |
| 是否按 DlvDate 正排？ | 是 | 已确认 |
| 是否暂不考虑物料齐套？ | 是 | 已确认 |
