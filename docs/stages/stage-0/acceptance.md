## 验收标准

### 必须通过项（阻断性）

以下条件全部满足，才可以进入阶段 1：

- [ ] **MVP 场景已明确**：文档中明确说明"第一版只排 EE Assembly 单场景（3F3~3F6）"。
- [ ] **每个必填字段有来源说明**：字段清单中，所有 `是否必填 = ✅` 的字段均记录了 AX 系统的对应字段名和表名。
- [ ] **UPPH 口径已确认**：明确使用线体 UPPH 还是人均 UPPH，并有书面记录。
- [ ] **样例数据可手工验证**：至少能手工走通一条订单的完整排产逻辑（筛单 → 匹配产线 → 计算产能 → 占用日历）。
- [ ] **订单筛选条件已确认**：业务团队确认了 Status / Pool / DlvDate 等筛选条件的口径。

### 建议通过项（非阻断性）

以下条件建议满足，未满足需记录风险：

- [ ] 样例数据集包含 EE 和 ESG 两类订单。
- [ ] 异常类型清单已与业务团队评审。
- [ ] 日历规则（周六/节假日/月末）已与运营团队确认。
- [ ] 数据库表结构设计已完成初稿（`data-objects` 页面）。

## 手工验证步骤

以下是对一条样例订单进行手工验证的参考步骤：

```text
1. 选取一条 Pool=SC_YBSC_F3、Category=EE 的订单
2. 确认其 FG Item Code 在 route_operations 中有路线
3. 找到 Assembly 工段对应的 ProcessType
4. 确认 3F3~3F6 中至少一条产线支持该 ProcessType
5. 从 capacity_param_inputs 中取该料号+产线的 UPPH
6. 从 work_calendar 中取 DlvDate 前后的工时安排
7. 计算：需要天数 = Qty / (UPPH × 每日工时)
8. 确认计算结果与预期相符，无数据缺失
```

若步骤中任一环节缺少数据，则说明阶段 0 未完成，需补充相应数据。
