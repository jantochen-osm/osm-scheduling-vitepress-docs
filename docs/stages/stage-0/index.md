# Stage 0 详细拆分：业务边界与数据底座确认

本文件夹用于承接 **阶段 0：业务边界与数据底座确认** 的详细执行方案。

阶段 0 的目标不是开发排产算法，而是把后续排产系统所依赖的 **业务范围、数据口径、样例数据、主数据、规则基线、异常口径、验收标准** 先固定下来。

::: tip 阶段 0 的核心判断
如果阶段 0 没有确认清楚，阶段 1 的 EE Assembly MVP 很容易出现：字段缺失、路线查不到、UPPH 口径不一致、产线规则冲突、排产结果无法解释等问题。
:::

## Stage 0 完成后应能回答的问题

- 第一版 MVP 到底只排什么？
- 订单从哪里来？字段含义是否统一？
- OSM 的 EE / ESG 分类字段是否可靠？
- 哪些 Pool 进入排产？哪些状态排除？
- ItemId（成品料号）如何查工艺路线？
- 工段名称如何标准化成 ProcessType？
- UPPH 是线体级、产品级、工段级还是人均级？
- 周六、周日、节假日、月底最后工作日如何计算工时？
- 缺数据、缺规则、缺产能时如何进入异常？
- 准备哪些样例数据可以支撑 Stage 1 MVP？

## 与 Stage 1 的关系

Stage 0 输出的数据样本、字段映射、规则基线、异常字典，将直接作为 Stage 1 的输入。

```text
Stage 0 输出
  ↓
字段清单 / 样例数据 / 规则基线 / 异常字典 / 验收清单
  ↓
Stage 1 EE Assembly 单场景 MVP
```
