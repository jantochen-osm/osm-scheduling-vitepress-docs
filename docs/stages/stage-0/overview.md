# 阶段 0 总览：业务边界与数据底座确认

## 阶段目标

阶段 0 是整个排产系统建设的**起点与基石**，不产出任何排产逻辑，只聚焦于以下三件事：

1. **明确系统排什么**：确认排产对象、排产粒度、纳入范围和暂不纳入范围。
2. **明确用什么数据排**：梳理所有必需的输入字段，确认数据来源与口径。
3. **准备可验证的数据样本**：收集至少 20~50 条真实或脱敏订单，使阶段 1 可以立即开始调试。

::: tip 核心原则
阶段 0 的产出是**文档与样本数据**，不是代码。阶段 0 未完成时，阶段 1 不应启动。
:::

## 排产对象与粒度

| 维度 | 确认内容 |
| --- | --- |
| 排产对象 | MO 制造订单 |
| 初始排产粒度 | MO + 工段 + 日期 + 产线 |
| 初始范围 | OSM 产线（EE + ESG 两大分类） |
| MVP 优先场景 | EE Assembly 单场景（3F3~3F6） |

## 阶段产出物清单

| 产出物 | 说明 |
| --- | --- |
| 业务边界确认文档 | 纳入/暂不纳入范围的列表 |
| 字段清单 | 每个字段的来源、口径、是否必填 |
| 规则清单 | 订单筛选、产线映射、优先级初稿 |
| 异常类型清单 | 初步识别的缺数、缺规则、冲突类型 |
| 样例数据集 | 20~50 条可供验证的订单样本 |
| MVP 场景确认 | 明确第一版只排哪个场景 |

## 与相邻阶段的关系

| 方向 | 说明 |
| --- | --- |
| 依赖 | 无前置阶段，从零开始 |
| 交付给阶段 1 | 样例数据、字段清单、MVP 范围确认文档 |
| 阶段 1 前置条件 | 阶段 0 所有产出物均已确认，字段来源清楚 |
