# 处理流程

## 阶段 0 工作步骤

阶段 0 是一个**调研与文档化**阶段，所有步骤的产出物都是文档或数据样本，不涉及代码开发。

```text
步骤 1：收集订单样例数据
步骤 2：收集工艺路线样例
步骤 3：梳理产线与工段映射关系
步骤 4：确认 UPPH 口径与数值来源
步骤 5：确认工作日历口径与规则
步骤 6：识别初始异常类型
步骤 7：输出字段清单、规则清单与异常清单
步骤 8：MVP 场景范围确认与签字
```

---

## 步骤详解

### 步骤 1：收集订单样例数据

**负责人**：数据/IT 团队  
**输出**：`mo_orders` 样例数据集（Excel 或 CSV）

- 从 AX 2012 或中间库导出 **20~50 条** 真实订单，或使用脱敏数据。
- 确保样本覆盖不同 Pool、不同 OSM_Category（EE / ESG / 未知）。
- 确保样本中包含 DlvDate 跨越多个月份的订单（测试日历占用逻辑）。
- 记录每个字段的来源表名和字段名（AX 侧）。

**验收条件**：
- [ ] 样本行数 ≥ 20 条
- [ ] 包含 EE 和 ESG 两类订单
- [ ] 包含至少 1 条字段缺失的"异常订单"（用于测试异常诊断）

---

### 步骤 2：收集工艺路线样例

**负责人**：数据/IT 团队 + 工程团队  
**输出**：`route_operations` 样例数据集

- 导出步骤 1 中订单对应的工艺路线数据。
- 确认路线中工序的 OperationCode 与排产系统定义的工段类型（ProcessType）如何对应。
- 记录"一个料号多条路线"的处理规则。

**验收条件**：
- [ ] 每条样例订单都能找到对应路线
- [ ] 工序到工段类型的映射表已确认

---

### 步骤 3：梳理产线与工段映射关系

**负责人**：生产运营团队  
**输出**：`production_lines` 主数据初稿

- 整理所有 OSM 产线的代码、楼层、分类（EE/ESG）。
- 确认每条产线支持哪些工段（Assembly / Package / SMT 等）。
- 标记哪些产线是试产线（不纳入正式排产）。
- 标记 ESG 专线的客户归属（4F1=Amazon / 4F4=Shure / 4F6=Jano-Life）。

**验收条件**：
- [ ] 所有 OSM 产线已列入清单
- [ ] 每条产线的 isSchedulable 状态已确认
- [ ] 试产线已明确标记

---

### 步骤 4：确认 UPPH 口径与数值来源

**负责人**：工程团队 + 生产运营团队  
**输出**：`capacity_param_inputs` 样例数据集

- 明确使用**线体 UPPH**（推荐）还是**人均 UPPH**。
- 收集步骤 1 样例订单中涉及产品的 UPPH 值。
- 确认 UPPH 是否按产线区分（同一产品在不同产线 UPPH 不同）。
- 确认效率系数是否需要配置。

**验收条件**：
- [ ] UPPH 口径（线体/人均）已确认并文档化
- [ ] 样例订单所有涉及料号的 UPPH 已收集

---

### 步骤 5：确认工作日历口径与规则

**负责人**：生产运营团队  
**输出**：`work_calendar` 初始数据 + 日历规则说明文档

- 确认正常工作日工时（默认 10h，是否有产线差异？）。
- 确认周六/周日/节假日的工时规则。
- 确认"节假日前一天"和"月末最后工作日"的短工时规则。
- 初步录入当年的法定节假日列表。

**验收条件**：
- [ ] 所有日期类型的默认工时已确认
- [ ] 当年法定节假日已录入
- [ ] 产线级日历例外的录入方式已确认

---

### 步骤 6：识别初始异常类型

**负责人**：系统设计团队 + 业务团队  
**输出**：异常类型清单（`schedule_exceptions` 设计初稿）

梳理排产过程中可能出现的异常，至少包括：

| 异常类型 | 说明 | 严重程度 |
| --- | --- | --- |
| 缺工艺路线 | 订单对应料号在 `route_operations` 中找不到路线 | ERROR |
| 缺 UPPH | 订单对应料号+产线+工段在 `capacity_param_inputs` 中找不到 UPPH | ERROR |
| 无可用产线 | 经规则匹配后，找不到满足条件的候选产线 | ERROR |
| OSM_Category 为空 | 订单无法分类为 EE 或 ESG | WARNING |
| DlvDate 为空 | 订单缺少交货日期，无法排产 | ERROR |
| 逾期风险 | 产能不足，预计完成日期超过 DlvDate | WARNING |

**验收条件**：
- [ ] 异常类型清单已与业务团队确认
- [ ] 每类异常的严重程度和建议处理方式已明确

---

### 步骤 7：输出字段清单与规则清单

**负责人**：系统设计团队  
**输出**：正式字段清单文档 + 规则清单文档

- 整合步骤 1~6 的所有字段确认结果，输出正式字段清单（含字段名、来源、口径、是否必填）。
- 整理订单筛选规则初稿（参考 `rules/osm-baseline`）。
- 整理产线映射规则初稿（EE/ESG 分类 + 专线规则）。

---

### 步骤 8：MVP 场景范围确认

**负责人**：项目负责人 + 业务负责人  
**输出**：MVP 范围确认文档（需相关方签字确认）

确认阶段 1 MVP 的精确范围：

| 确认项 | 确认结果 |
| --- | --- |
| MVP 只排哪个场景 | EE Assembly（3F3~3F6） |
| MVP 使用哪些订单 | Pool=SC_YBSC_F3 + Category=EE |
| MVP 验收方式 | 手工验证至少 1 条订单的完整排产过程 |
| 何时可以进入阶段 1 | 上述步骤 1~7 全部验收通过 |
