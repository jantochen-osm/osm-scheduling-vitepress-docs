# 业务边界确认

## 纳入范围

### 排产对象

- **MO 制造订单** 是唯一的排产对象。
- 排产粒度：`MO + 工段 + 日期 + 产线`。

### 产线范围

初始纳入 OSM 管辖的所有 EE 与 ESG 产线（见下表）：

| 楼层 | 产线代码 | 类别 | 备注 |
| --- | --- | --- | --- |
| 1F | 1F1、1F2 | EE Package | 正式纳入 |
| 1F | 1F3 | EE Package / 试产 | 初期仅试产，暂不纳入正式排产 |
| 3F | 3F1 | EE 半成品 / 特定 Assembly | 正式纳入 |
| 3F | 3F2 | EE 特定产品全工段 | 正式纳入 |
| 3F | 3F3~3F6 | EE Assembly | 正式纳入，MVP 优先场景 |
| 4F | 4F1 | ESG Amazon 专线 | 正式纳入 |
| 4F | 4F2 | ESG Chicha | 正式纳入 |
| 4F | 4F3、4F5 | ESG 试产 | 初期仅试产，暂不纳入正式排产 |
| 4F | 4F4 | ESG Shure 专线 | 正式纳入 |
| 4F | 4F6 | ESG Jano-Life 专线 | 正式纳入 |
| 4F | 4F7、4F8 | ESG 待确认 | 需业务确认后决定 |

### MVP 优先场景

阶段 1 的 MVP 优先聚焦 **EE Assembly 单场景**：

- 产线：3F3、3F4、3F5、3F6
- 工段：Assembly（单工段）
- 订单筛选条件：`OSM_Category = EE` + `Pool IN (SC_YBSC_F3, SC_YBSC_HT)`

### 核心字段确认项

需确认以下字段在 AX 2012 / 中间库中的实际字段名和口径：

| 逻辑字段 | 含义 | 是否必填 |
| --- | --- | --- |
| MO | 制造订单号 | ✅ 必填 |
| FG Item Code | 成品料号 | ✅ 必填 |
| Qty | 计划数量 | ✅ 必填 |
| DlvDate | 交货日期 | ✅ 必填 |
| Status | 订单状态 | ✅ 必填 |
| Pool | 订单池 | ✅ 必填 |
| OSM_Category | EE / ESG 分类 | ✅ 必填 |
| Customer | 客户 | 建议有 |
| Brand | 品牌 | 建议有 |
| RouteId | 工艺路线号 | ✅ 必填 |

---

## 暂不纳入范围

以下能力不在阶段 0（乃至阶段 1）中实现，避免过早复杂化：

| 暂不纳入内容 | 说明 |
| --- | --- |
| 复杂排产算法 | 阶段 0 仅确认数据与边界，不开发算法 |
| 人工拖拽 / 手动调度界面 | 阶段 5 才引入人工干预能力 |
| 物料齐套 / BOM 展开 | 不在 OSM 排产系统初始范围内 |
| 多 BOM 版本管理 | 不在初始范围内 |
| 试产产线正式排产 | 1F3、4F3、4F5 仅在试产场景中使用 |
| 4F7、4F8 产线 | 待业务确认后再定 |

::: warning 注意
暂不纳入范围的内容应在阶段 0 产出的"业务边界确认文档"中明确记录，并由相关业务负责人确认签字，避免后续范围蔓延。
:::
