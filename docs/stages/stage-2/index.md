# 阶段 2：EE 多工段与完整 EE 规则扩展

## 目标

从单 Assembly 扩展到 EE Package、半成品、3F2 特定产品全工段，并加入工段顺序。

## 纳入范围

- 1F1~1F2 Package。
- 1F3 试产线排除正式单。
- 3F1 半成品及 FA014A02 Assembly。
- 3F2 特定 FG Item Code 全工段。
- 3F3~3F6 Assembly。
- 多工段按 OperationSeq 顺序排产。

## 暂不纳入范围

- ESG。
- 试产订单正式排产。
- 分批流转。
- 多线并行。

## 输入数据

- 完整 EE 工艺路线。
- ProcessType 与产线映射规则。
- 3F2 特定产品 FG Item Code 清单。
- 3F1 特殊规则。
- Package UPPH。

## 处理流程

```text
1. 在阶段 1 基础上读取完整工艺路线
2. 按 OperationSeq 排序
3. 逐工段匹配产线规则
4. 先处理专线规则，例如 3F2
5. 再处理工段规则，例如 Package / Assembly
6. 前工段整单完成后，后工段才允许开始
7. 生成多工段日明细
8. 输出跨工段开始/完成日期
```

## 输出结果

- 多工段排产结果。
- 工段顺序校验结果。
- 专线规则命中日志。
- EE 规则异常清单。

## 验收标准

- Assembly 与 Package 不出现顺序倒置。
- 3F2 专线产品优先排 3F2。
- 1F3 不承接正式订单。
- 缺工段顺序进入异常。

## 与下一阶段的衔接

阶段 2 完成后，系统已经具备“多工段排产框架”，阶段 3 扩展 ESG 时只需要增加 Line Group 和专线规则，不需要重写排产核心。

## 主要风险与控制

- 工艺路线工段命名不统一：建立 ProcessType 标准字典。
- 3F2 专线规则冲突：定义规则优先级和冲突异常。
