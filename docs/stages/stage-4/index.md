# 阶段 4：日历例外、异常诊断与可解释性增强

## 目标

让排产结果更可信、可解释、可诊断，支持周六、节假日前、月底最后工作日、停线、加班等日历例外。

## 纳入范围

- 周六 8h。
- 周日/法定假日 0h。
- 大型节假日前一天 8h。
- 每月最后一个工作日 8h。
- 产线级停线/加班。
- 异常分类。
- 规则命中日志。

## 暂不纳入范围

- 复杂换线损耗。
- 物料齐套。
- 自动优化算法。

## 输入数据

- 工作日历。
- 节假日清单。
- 产线级例外日历。
- 异常类型字典。
- 规则优先级配置。

## 处理流程

```text
1. 计算基础日历工时
2. 应用通用例外规则
3. 应用产线级例外规则
4. 每次产能计算记录使用的日历来源
5. 排产过程记录规则命中 trace
6. 异常按订单/工艺/规则/产能分类
7. 汇总逾期、瓶颈、负荷率
```

## 输出结果

- 日历例外生效后的排产结果。
- 异常诊断清单。
- 规则命中日志。
- 产线负荷率。
- 瓶颈工段。
- 逾期订单清单。

## 验收标准

- 同一天多种日历规则命中时按优先级处理。
- 每条异常都有类型、原因、建议动作。
- 每条排产结果能解释为什么排到该线和该日期。

## 与下一阶段的衔接

阶段 4 形成业务可用的可解释排产系统。阶段 5 将引入人工干预，使计划员可以调整并形成计划版本。

## 主要风险与控制

- 日历规则冲突：定义固定优先级。
- 异常太多影响使用：区分阻断异常和提醒异常。
