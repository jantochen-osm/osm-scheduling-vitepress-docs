# 阶段 5：人工干预与计划版本管理

## 目标

支持计划员对自动排产结果进行调整、锁定、重排，并形成可追踪的计划版本。

## 纳入范围

- 锁定订单。
- 锁定产线。
- 调整优先级。
- 手工改日期。
- 临时停线。
- 临时加班。
- 异常放行。
- 排产版本对比。

## 暂不纳入范围

- 自动拖拽甘特图高级交互。
- 复杂优化算法自动替代人工判断。

## 输入数据

- 自动排产结果。
- 人工调整记录。
- 锁定规则。
- 计划版本号。

## 处理流程

```text
1. 基于某次 runId 生成可调整计划版本
2. 计划员对订单/工段/产线/日期进行锁定或调整
3. 保存人工调整记录
4. 重排时保护已锁定结果
5. 对未锁定部分重新计算
6. 输出版本差异
7. 计划员确认发布
```

## 输出结果

- 计划版本。
- 人工调整日志。
- 锁定结果。
- 重排结果。
- 版本差异报告。

## 验收标准

- 重排不会覆盖已锁定结果。
- 每次人工修改有记录。
- 能比较两个版本的订单完成日和产线负荷差异。

## 与下一阶段的衔接

阶段 5 完成后，系统可作为计划员日常排产辅助工具。阶段 6 再扩展物料、人力、换线、多线并行等高级约束。

## 主要风险与控制

- 人工调整破坏工段顺序：调整后重新校验顺序约束。
- 版本混乱：每次发布必须绑定唯一 planVersion。
