# 阶段 6：高级约束与优化能力扩展

## 目标

在规则排产稳定后，引入物料齐套、人力动态、换线损耗、多线并行、分批流转和优化算法。

## 纳入范围

- 物料齐套状态。
- 动态人力计划。
- 换线损耗。
- 多线并行。
- 分批流转。
- 负荷均衡。
- 逾期最小化。

## 暂不纳入范围

- 不建议在阶段 1~3 过早引入。

## 输入数据

- 库存/采购/齐套数据。
- 人力计划。
- 产品族/换线矩阵。
- 历史生产效率。
- 计划评价指标。

## 处理流程

```text
1. 引入物料可用日期
2. 订单最早开工日期受物料约束
3. 引入每日人力影响产能
4. 引入产品切换损耗
5. 支持同一工段多线并行或分批流转
6. 引入目标函数，例如逾期最少、负荷均衡
7. 与规则排产结果对比评估
```

## 输出结果

- 高级约束排产结果。
- 优化前后对比。
- 逾期改善报告。
- 产能利用率报告。

## 验收标准

- 优化结果不能违反硬约束。
- 结果相比规则排产有可量化改善。
- 计划员仍可解释关键决策原因。

## 与下一阶段的衔接

阶段 6 是长期演进方向，不应影响前面版本稳定上线。高级能力应以插件式方式加入排产引擎。

## 主要风险与控制

- 优化黑盒导致业务不信任：保留规则命中与优化解释。
- 数据质量不足：没有可靠数据不启用对应约束。
